CSS, lists and favicons

Many more HTML notions, tags, and general good practices will be covered throughout this document. "Favicon" was chosen as the title because it's short and interesting. We will touch on basic CSS terminology and semantics, HTML lists (both ordered and unordered), and the two main categories of display elements — inline and block.

HTML Lists

We will start with lists because they are a straightforward and fundamental topic. Additionally, we've already used them in previous explanatory pages. There are two main types of lists in HTML: ordered and unordered. Ordered lists number their items; unordered lists use bullet points. Unordered and ordered lists are declared with <ul> and <ol>, respectively. List items are declared with the <li> tag. 

<!-- This is an unordered list -->
<ul>
<li>First item</li>
<li>Second item</li>
<li>Third item</li>
</ul>

<!-- This is an ordered list -->
<ol>
<li>First item</li>
<li>Second item</li>
<li>Third item</li>
</ol>

There is also a special type of list called a definition list, used to define terms and their corresponding definitions. Definition lists are declared with <dl>, while terms and definitions are declared with <dt> and <dd>, respectively. An example is shown below.

<!-- This is a definition list -->
<dl>
<dt>HTML</dt>
<dd>Hypertext Markup Language</dd>
<dt>CSS</dt>
<dd>Cascading Style Sheets</dd>
</dl>

Inline and block elements

Before jumping straight into CSS, it's worth presenting the concept of inline vs. block elements in HTML. This property specifies an element's default display behavior in the browser. Block elements are, by default, full-width. Browsers generally add vertical spacing before and after block elements. The most common block elements are <div> and <p>.

Inline elements, on the other hand, are only as wide as their content and do not force line breaks. The most common inline elements are <span> and <a>. They are often used to color text elements that are components of paragraphs, such a use can be seen in the previous sentence.

For a thorough list of both inline and block elements, please consult the HTML documentation. MDN docs.

A brief look into CSS

We will now take a look at CSS and how it can improve the appearance of our web pages. CSS stands for Cascading Style Sheets. It is called "cascading" because styles applied to a parent element can be inherited by its children. A simple example is the centering used in this document. CSS can change text alignment, colors, borders, dimensions, and much more.

Using CSS

There are three main ways to use CSS in an HTML document:

  • Inline CSS — apply style to a single element using the style attribute.
  • Internal CSS — define CSS rules inside a <style> tag in the <head> for use within the same document.
  • External CSS — place styles in a separate .css file and link it with <link> in the <head>.

Here are some commonly used CSS properties:

  • color — sets the text color.
  • background-color — sets the element's background color.
  • text-align — sets horizontal alignment of text within an element.
  • font-size — sets the font size.
  • font-family — sets the font type.
  • border — sets border properties of an element.
  • width — sets the element width.
  • height — sets the element height.

A few examples are shown below. The relevant snippet that deals with style is highlighted.

An exhaustive list of CSS properties and documentation is available at the CSS Reference (MDN). 

<p style="color:blue; text-align:center;">This is an example of inline CSS</p>

For internal and external CSS, we will supply both the additional code and the steps to apply it. For internal CSS, define rules inside a <style> tag in the <head>. For external CSS, create a .css file and link it using <link> in the <head>. Below is an example of internal CSS. 

<style>
.title-center {
    display: block;
    text-align: center;
}

.para-center {
    display: block;
    margin-left: auto;
    margin-right: auto;
    width: 50%;
}

pre.example-box {
    display: block;
    text-align: center;
    margin-left: auto;
    margin-right: auto;
    width: fit-content;
    border: 1px solid black;
}

p {
    font-family: Arial, sans-serif;
    font-size: 14px;
}
</style>

As we can see, a CSS rule has a selector and a set of properties. Selectors can be class names (prefixed with a period, e.g., .my-class) or IDs (prefixed with #, e.g., #my-id). Use classes to apply the same style to multiple elements; use IDs when a unique style is needed for a single element.

Linking an external CSS document is achieved with the <link> tag in the <head> section. Example: 

<link rel="stylesheet" type="text/css" href="styles.css">

Favicons

At last, we will cover favicons and their use. First things first, a favicon is the miniature icon or image that is displayed in the tab's title. Similarly to how we may link a CSS style file to our HTML document, we use a favicon by referencing a file name in the link tag. An example of favicon usage can be found below. 

<link rel="icon" type="image/png" href="favicon.png">

Favicons can be in various formats, such as PNG, ICO, or SVG. The type attribute specifies the image format, while the href attribute points to the favicon file's location. It's common practice to place the favicon file in the root directory of your website for easy access.
