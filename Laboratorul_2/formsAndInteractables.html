Forms and Interactables


Prelude

This document will teach us on what forms and interactables are and how we may use them to enhance the user experience. Furthermore, it will be presented why and how forms represent the principal method of web-page user data acquisition. It is through forms that a user may interact with the data acquisition and processing mechanisms that sit behind the front of our web-page. It is forms precisely that we encounte whenever a web-site prompts us to commit our data for various purposes such as payment procedures, registration and so much more.

It is also wise to view forms as compound structures that achieve their role through the use of various interactable objects. These interactable objects are HTML items a user will interact with, such as buttons, drop-down lists and text boxes. In order to study and then use forms we first have to some of the most important objects we can make use of. This is precisely what the following chapters will try to achieve.

Buttons and text boxes

Buttons are relatively simple structures with some interesting fiels we'll go over. A button can be declared by using the button tag.


<button> Press Me

The result of this tag can be seen below, it's quite simple.

There are many CSS style options specific to buttons that we may use in improving their style, such as:

  • button.buttonClass:hover defines what shall occur when our cursor hovers over the button.
  • transition-duration defines how long it takes for our button to change color as specified in the hover property.
  • cursor defines how our cursor will look when the mouse hovers over the button's box. There are many options, such as auto, pointer, default, move, not-allowed and several others.

We can see the effects and their length of duration by hovering over the buttons above. In the context of forms, there is another attribute of buttons that we may use. That is, the button type option. It has two possible values, submit and reset. Reset may be used to clear the set context of the form's data fields while submit is generally used to submit the personalized data to a back-end processing pipeline, however simple.

Forms and Inputs

As has been elaborated before, forms are no more than entities defined by the sum of their interactable components. These components bear the name of inputs and they can be of many types, as follows:

  • <input type="text"> are the basic elements for raw text submissions, regardless of any format unless so enformed by the back-end or through web-page scripts.
  • <label> is an element with a fixed descriptive function, used for the annotation of other interactables.
  • <input type="radio"> is a multiple option and (generally) single choice submission data acquisition input.
  • <input type="checkbox"> is generally used to answer questions that have well defined and fixed options.
  • <input type="submit"> is a button used for form submission.
  • <input type="button"> is a general button that can serve many functions, often times it can even be used to generate in-page pop-up secondary forms.
