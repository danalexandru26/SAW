Tables, classes and identifiers

In this document we will discuss some of the most important HTML tags and properties. Tables serve us with an ubiquitous way of organizing and displaying data of almost any kind. There are scarcely any limits to what a table can functionally display and do. The limitation sits enirely with how we organize, store and display our data.

Furthermore, we will also dive into the intricacies of HTML classes that we may associate to tags, identifiers and some finer details on div block elements. Such chapters will also serve as an introduction to more advanced web development and design concepts such as the use of javascript and CSS.

Tables

Tables serve as the primary mode of data display in a variety of fields. They have become a term that's virtually interchangeable with data engineering and analysis. Because of this fact, learning to declare and properly use tables is an ability as important as it is easy to develop. The content that follows in this section will describe the proper way of table instantiation, styling and use.

In order to declare a table, we shall use the <table> tag. The subsequent tags that will instantiate the content rows and headers are in succession the following: <tr>, <th>, <td>. They stand for table row, table header and table data respectively. Below, we can find a sample example of how a table is declared. 

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
    <th>City</th>
    </tr>
    <tr>
        <td>Alice</td>
        <td>30</td>
        <td>New York</td>
    </tr>
    <tr>
        <td>Bob</td>
        <td>25</td>
        <td>Los Angeles</td>
    </tr>
</table>

There are several CSS style options that were used to properly display our table and the data therein. Without any stylistic modifications, the table will be raw, unbordered and devoid of any visual enhancements. The CSS options that were used to aid in displaying our table are as follows:

  • Borders are defined using the border style option.
    • 
border: 1px solid black;
  • The padding that sits between the text's extremities and the table cell's border are defined with the padding attribute. 
    • 
padding: 10px;
  • How the borders of our table cell function is controlled by the border-collapse attribute. The individual cell borders will collapse when using this attribute with the collapse option, giving our table an unified centralized look.
    • 
border-collapse: collapse;
  • In order to alternate the color of our table rows we do the following:
    • 
tr:nth-child(even) {
    background-color: #f2f2f2;
}

IDs and their usage

IDs are unique identifiers for specific elements on our web page. IDs allow us to exercise fine grained control on how certain tags are displayed and addressed. In addition, by using these identifiers, we are enabled to transform our page in a more interactive manner through the use of javascript snippets. IDs are declared by using the id attribute.


<h1 id="headerStyle1">

In order to define a style that is to be used for only a specific ID, we shall use the syntax described below. Notice the # prefixed before the style name proper. It signifies that this style is to be used for the element assigned to the ID that bears the same name.


<style>
    #headerStyle1{
        display: block;
        text-align: center;
    }
<style>

Identifiers can also be useful in controlling certain HTML forms items, but this will be addressed in another document. Interestingly enough, they can also be used in defining jumps to a document section for which the ID was defined.


<a href="#headerStyle1"> Jump to the specified ID.
